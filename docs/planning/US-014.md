## Analisi requisiti

**Analyst**

Il pulsante "Invita" è esclusivo del creatore: il confronto `circle.ownerId === currentUserId` deve avvenire lato frontend (il `CircleSummary` già espone `ownerId`; `AuthService.getCurrentUserId()` già decodifica il JWT). Il backend rafforza con un check `circle.OwnerId == userId` sull'endpoint e risponde 403 se l'utente non è il creatore.

Il token di invito è già modellato nell'entità `Circle.JoinCode` (campo `string?` esistente, riservato per questo flusso secondo `CLAUDE.md`). **Non serve migrazione**: si popola il campo la prima volta che viene chiamato `GET /circles/{id}/invite-link` (lazy generation con `Guid.NewGuid().ToString("N")`). Tutte le chiamate successive allo stesso circolo restituiscono lo stesso token (stabile per design). L'open question "va rigenerato?" è risolta dalla storia stessa: "generato una volta sola, stabile".

Il frontend costruisce l'URL completo come `${window.location.origin}/join?token=${inviteToken}`. Il pulsante "Copia" usa `navigator.clipboard.writeText()`. Il pulsante "Email" usa un link `mailto:` standard; il corpo del messaggio contiene l'URL. La dialog non è un componente di libreria (non c'è Angular Material): si realizza come overlay CSS inline nella `my-circles` page, con un nuovo `InviteDialogComponent` standalone che riceve `circleId` e `circleName` come `@Input`.

---

## Soluzione tecnica

**Architect**

**Approccio:** aggiungere un solo endpoint backend (che popola `Circle.JoinCode` lazily), un metodo nel service, e un componente dialog inline nella pagina `my-circles`.

**Decisioni chiave:**
- Riuso `Circle.JoinCode` senza migrazione. Alternativa (campo `InviteToken` separato) aggiunge complessità inutile.
- Token generato lazy: il creatore non paga nulla alla creazione del circolo; il token viene creato solo se richiesto.
- La dialog è inline (overlay) nella `my-circles` page anziché una route separata: la UX vuole un'azione rapida dalla lista circoli, non una navigazione.

**Componenti toccati:**

| Layer | File | Modifica |
|---|---|---|
| Backend | `src/Golp.Api/Endpoints/CircleEndpoints.cs` | + `GET /{id}/invite-link` |
| Test BE | `src/Golp.Tests/Integration/InviteLinkEndpointTests.cs` | nuovo file |
| Service FE | `frontend/.../circles/circle.service.ts` | + `getInviteLink()` |
| Componente | `frontend/.../circles/invite-dialog/` | nuovo (ts + html + scss) |
| Host | `frontend/.../circles/my-circles/my-circles.component.*` | + btn + dialog mount |

**Contratto API:**
```
GET /circles/{circleId}/invite-link
Authorization: Bearer <jwt>
→ 200: { inviteToken: "abc123..." }
→ 401: non autenticato
→ 403: { error: "Solo il creatore può generare il link di invito" }
→ 404: { error: "Circolo non trovato" }
```

**Rischi:**
- `navigator.clipboard.writeText()` richiede HTTPS o localhost. In produzione Azure (HTTPS) va bene; in dev va bene perché localhost. Nessun fallback necessario.
