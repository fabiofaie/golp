## Analisi requisiti

US-042 aggiunge alla pagina di successo post-registrazione partita un meccanismo di condivisione del link di conferma per ogni partecipante non-creatore. Tre modalità si escludono a cascata: (1) WhatsApp diretto se il partecipante ha numero di telefono, (2) Web Share API se non ha numero ma il browser la supporta, (3) campo testo con copia manuale come fallback universale.

Il requisito AC1 implica che la response HTTP della creazione partita deve già includere i dati token per i 3 non-creatori (userId, name, phone, tokenUrl) — il backend li genera già atomicamente ma non li restituisce nel body. L'estensione della response è il cambiamento più rilevante lato backend, ma rimane minima: una query sincrona post-save per nome+phone dei recipient.

Edge case rilevanti: creatore non ha share button (nessun token per sé stesso); ospiti phone-only ricevono il pulsante WhatsApp (il token esiste, il phone è in `User.Phone`); `navigator.share()` annullato dall'utente va catturato silenziosamente; phone in formato locale (`340 1234567`, `+39 340...`) normalizzato client-side strip non-digit; Quick Match ha stesso requisito (flusso alternativo che genera partita con stesso meccanismo token).

Requisiti impliciti: il `ShareConfirmComponent` deve essere standalone e riutilizzabile (US-043 lo riusa); feature detection di `navigator.share` sicura in SSR/test: `typeof navigator !== 'undefined' && 'share' in navigator`.

---

## Soluzione tecnica

**Approccio:** estensione minimale del contratto API esistente. I token sono già generati + salvati atomicamente prima del `SaveChangesAsync`. Serve solo una query sincrona post-save sui 3 recipient per nome+phone, poi il dato viene aggiunto al body `201 Created`. Lato frontend: un nuovo componente Angular standalone `ShareConfirmComponent` con le 3 modalità; i componenti `record-match` e `quick-match` passano a uno success state interno invece di navigare.

**Decisioni chiave:**

| Decisione | Scelta | Alternativa scartata |
|---|---|---|
| Dati token nella response | Aggiunti al body `201 Created` | Endpoint GET separato `/matches/{id}/tokens` |
| Motivo | Zero round-trip extra; frontend ha tutti i dati in memoria | Extra fetch introduce latenza e complessità |
| Phone normalization | Client-side, `.replace(/\D/g, '')` | Normalizzazione server-side |
| Motivo | Strip non-digit converte `+39 340 1234567` → `393401234567` correttamente per wa.me senza dipendere da country code | |
| Success state | In-component flag `matchCreated` | Nuova route dedicata |
| Motivo | Dati già in memoria, no re-fetch, URL pulita | |
| Share component | Standalone Angular `@Input` | Inline template duplicato in ogni form |
| Motivo | Riutilizzabile in US-043, testabile in isolamento | |

### TASK-4 — Frontend: crea `ShareConfirmComponent` ✅ (2026-07-01 17:30)

**File:** `frontend/golp-app/src/app/circles/share-confirm/share-confirm.component.ts` + `.html`

```ts
@Component({
  selector: 'app-share-confirm',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './share-confirm.component.html',
})
export class ShareConfirmComponent {
  @Input() links: ConfirmationLink[] = [];
  @Input() sport = '';
  @Input() circleName = '';

  readonly hasShare =
    typeof navigator !== 'undefined' && 'share' in navigator;

  waUrl(link: ConfirmationLink): string {
    const phone = link.phone!.replace(/\D/g, '');
    const msg = encodeURIComponent(
      `Ciao ${link.name}! Ho registrato una partita di ${this.sport} nel circolo ${this.circleName}. Conferma il risultato qui: ${link.tokenUrl}`
    );
    return `https://wa.me/${phone}?text=${msg}`;
  }

  async shareLink(link: ConfirmationLink): Promise<void> {
    try {
      await navigator.share({
        title: 'Conferma partita',
        text: `Ciao ${link.name}! Ho registrato una partita di ${this.sport} nel circolo ${this.circleName}. Conferma il risultato qui:`,
        url: link.tokenUrl,
      });
    } catch { /* user cancelled */ }
  }

  copyToClipboard(url: string): void {
    navigator.clipboard.writeText(url);
  }
}
```

Template: per ogni link:
- Se `link.phone`: `<a [href]="waUrl(link)" target="_blank">Invia via WhatsApp a {{ link.name }}</a>`
- Altrimenti se `hasShare`: `<button (click)="shareLink(link)">Condividi link a {{ link.name }}</button>`
- Altrimenti: `<input [value]="link.tokenUrl" readonly> <button (click)="copyToClipboard(link.tokenUrl)">Copia</button>`

**Acceptance:** tutti e 3 i rami visibili con i giusti input; `waUrl()` costruisce URL corretto.
