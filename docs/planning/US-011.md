## Analisi requisiti

**AC e comportamenti concreti:**
1. **Miglior compagno** — per ogni partner con cui ho giocato ≥3 partite `confirmed` nel circolo → `wins_together / total_together`; miglior compagno = max win-rate. Se nessun partner raggiunge la soglia → `null`.
2. **Avversario più ostico** — per ogni avversario contro cui ho giocato ≥3 partite confirmed → `my_wins_vs / total_vs`; avversario più ostico = min win-rate. Se nessuno raggiunge soglia → `null`.
3. **Solo partite confirmed** — `Status = "confirmed"` e `CircleId = circleId`.
4. **Per-circolo** — `circleId` route param; dati diversi per circoli diversi.
5. **Soglia N=3** — partner/avversario con < 3 partite escluso dal calcolo anche se win-rate 100%.
6. **Nessuna partita → stato vuoto** — `bestPartner: null, toughestOpponent: null`, nessun errore.

**Definizioni precise per match 2v2:**
- Io sono `userId`. In ogni match:
  - `Team1Player1Id == userId` OR `Team1Player2Id == userId` → sono Team1; partner = altro Team1; avversari = Team2Player1 + Team2Player2; win = `WinnerTeam == 1`.
  - `Team2Player1Id == userId` OR `Team2Player2Id == userId` → sono Team2; partner = altro Team2; avversari = Team1Player1 + Team1Player2; win = `WinnerTeam == 2`.
- Ogni match contribuisce a **1 partner** e **2 avversari** (separati).

**Edge case:**
- Parità win-rate tra partner → tie-break: `gamesTogether DESC`, poi `partnerId ASC`.
- Parità win-rate tra avversari → tie-break: `gamesAgainst DESC`, poi `opponentId ASC`.
- Utente non membro del circolo → 403.
- Tutti i partner/avversari < 3 partite → rispettivi null indipendenti (bestPartner null non implica toughestOpponent null).

**Requisiti impliciti:**
- Nuovo endpoint `GET /circles/{circleId}/stats/me`.
- Calcolo in-memory dopo fetch (2 query EF: matches come team1, matches come team2).
- Nomi di partner/avversari risolti da `Users`.
- Route Angular `/circles/:circleId/stats`; link "Stats" da MyCircles.

---

## Soluzione tecnica

**Approccio:** Nuovo endpoint `GET /circles/{circleId}/stats/me`. Due query EF (matches come team1 + matches come team2) per evitare OR su 4 colonne. Aggregazione in-memory: per ogni match determina partner e avversari, accumula stats, filtra per soglia N=3, seleziona best/toughest.

**Componenti toccati:**

| Layer | Componente | Modifica |
|-------|-----------|----------|
| Backend | `StatsEndpoints.cs` | **new**: `GET /circles/{circleId}/stats/me` |
| Backend | `Program.cs` | registra `MapStatsEndpoints` |
| Frontend | `circle.service.ts` | aggiunge `PlayerStatSummary`, `CircleStatsResponse`, `getMyStats()` |
| Frontend | `circle-stats/` | **new component** `CircleStatsComponent` (ts + html + scss) |
| Frontend | `app.routes.ts` | aggiunge route `/circles/:circleId/stats` |
| Frontend | `my-circles.component.html` | aggiunge link "Stats" su ogni card |

**Contratto API:**
```
GET /circles/{circleId}/stats/me
Authorization: Bearer <jwt>
→ 200 {
    "bestPartner": {
      "userId": "...", "name": "...", "winRate": 0.75, "gamesTogether": 8
    } | null,
    "toughestOpponent": {
      "userId": "...", "name": "...", "winRate": 0.25, "gamesAgainst": 5
    } | null
  }
→ 403  non membro del circolo
→ 404  circolo non trovato
→ 401  non autenticato
```

**Logica aggregazione (schema in-memory):**
```csharp
// 2 query separate + concat per evitare OR su 4 colonne
var asTeam1 = await db.Matches
    .Where(m => m.CircleId == circleId && m.Status == "confirmed"
             && (m.Team1Player1Id == userId || m.Team1Player2Id == userId))
    .ToListAsync();
var asTeam2 = await db.Matches
    .Where(m => m.CircleId == circleId && m.Status == "confirmed"
             && (m.Team2Player1Id == userId || m.Team2Player2Id == userId))
    .ToListAsync();
var allMatches = asTeam1.Concat(asTeam2).ToList();

// Per ogni match: determina partner, avversari, win
var partnerStats = new Dictionary<Guid, (int Total, int Wins)>();
var opponentStats = new Dictionary<Guid, (int Total, int Wins)>();

foreach (var m in allMatches) {
    bool iAmTeam1 = asTeam1.Contains(m);
    bool iWon = (iAmTeam1 && m.WinnerTeam == 1) || (!iAmTeam1 && m.WinnerTeam == 2);
    Guid partner = iAmTeam1
        ? (m.Team1Player1Id == userId ? m.Team1Player2Id : m.Team1Player1Id)
        : (m.Team2Player1Id == userId ? m.Team2Player2Id : m.Team2Player1Id);
    Guid[] opponents = iAmTeam1
        ? [m.Team2Player1Id, m.Team2Player2Id]
        : [m.Team1Player1Id, m.Team1Player2Id];
    // accumula partnerStats[partner] e opponentStats[opp]
}

// Filtra N>=3, calcola winRate, prendi best/worst
```

**Decisioni chiave:**
- **2 query EF + concat** — più leggibile di un OR su 4 colonne; dataset piccolo rende il concat in-memory trascurabile.
- **Nessuna tabella statistiche** — sempre fresh dai match, nessun dato derivato da mantenere sincronizzato.
- **N=3 hardcoded** — MVP; parametrizzazione out of scope (PRD).
- **Partner e avversari trattati separatamente** — la soglia N=3 si applica indipendentemente ai due conteggi.

**Rischi:**
- US-005 non DONE: nessuna partita confirmed → statistiche sempre vuote.
- `asTeam1.Contains(m)` per determinare `iAmTeam1` è O(n²) se la lista è grande — per MVP (< 100 partite per circolo) accettabile; alternativa: usare un `HashSet`.
