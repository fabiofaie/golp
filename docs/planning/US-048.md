## Analisi requisiti

Il form "registra partita" deve conoscere `allowsSingles` per lo sport del circolo. `CircleSummary` non espone questo campo — serve una chiamata a `GET /sports` in `ngOnInit` per recuperarlo. `SportConfig` in `circle.service.ts` va aggiornato con `allowsSingles: boolean`.

In modalità singolo gli slot attivi sono indici `[0]` e `[2]` (uno per squadra). Il submit costruisce `team1=[slots[0]]`, `team2=[slots[2]]` e passa `isSingles: true`. Validazione loop e `otherSlotIndexes` vanno resi condizionali.

Il badge "1v1" in lista si ricava senza campo extra: se `m.team1.length === 1` la partita è singolo — il backend già restituisce l'array corretto. `confirmDots()` usa attualmente `length: 4` hard-coded; va reso dinamico con `m.team1.length === 1 ? 2 : 4`. Il dettaglio partita è già safe: il template usa `@for` su `team1`/`team2`.

---

## Soluzione tecnica

**Approccio:** Modifiche chirurgiche a 4 file TypeScript + 2 HTML. Nessuna modifica backend. Nessun nuovo servizio né componente.

| File | Cambiamento |
|---|---|
| `circle.service.ts` | `SportConfig` + `allowsSingles: boolean` |
| `match.service.ts` | `CreateMatchRequest` + `isSingles?: boolean` |
| `record-match.component.ts` | fetch sport, flag `isSingles`, submit branch, slot singoli |
| `record-match.component.html` | toggle visivo, slot condizionali, testo generico |
| `circle-match-history.component.ts` | `isSingles(m)` helper, `confirmDots` dinamico |
| `circle-match-history.component.html` | badge "1v1", label N dinamica |

**Decisione — `allowsSingles`:** chiamata a `getSports()` in `ngOnInit` di `RecordMatchComponent`, in parallelo con i call esistenti. Aggiunta `allowsSingles: boolean = false` nel componente. Alternativa (aggiungere il campo a `CircleSummary`) richiederebbe modifica backend fuori scope.

**Decisione — slot singolo:** slot attivi `[0,2]` in singolo; validazione loop e `otherSlotIndexes` usano `activeSlotIndexes()`. Nessun refactor del meccanismo ospite (funziona con qualsiasi indice).

## Rischi aperti

- `getSports()` è chiamata extra nel form. Se il backend è lento, potrebbe introdurre un flash visivo prima che il toggle compaia. Mitigation: `allowsSingles` default false → form parte in modalità doppio, toggle appare dopo il caricamento.
- Se `circle` è null (non trovato) prima del completamento di `getSports()`, il confronto `circle.sport` darebbe errore → verificare ordine subscriptions in `ngOnInit`.
