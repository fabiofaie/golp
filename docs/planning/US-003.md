## Analisi requisiti

AC1-AC5 mappano su tre operazioni distinte: *discovery* (trovare un circolo — prerequisito implicito non scritto negli AC), *join* (creazione membership), e *members list* (verifica isolamento). Il data model è già pronto: `Circle.IsPrivate` default=false, `CircleMembership.Rating=1000` come default, relazioni EF Core già definite in US-002.

**Open question risolta:** `IsPrivate`+`JoinCode` sono nel modello ma non nel DTO di US-002, marcati "intenzionale per future US". Free join per circoli pubblici (`IsPrivate=false`) è l'approccio MVP; US-003 esclude esplicitamente approvazione manuale. Decisione: `POST /circles/{id}/join` senza codice per circoli `IsPrivate=false`. Join su circoli privati → 403 (protegge infrastruttura futura).

**Requisito implicito incluso:** per AC1 ("unirsi a un circolo esistente") serve trovarlo prima. Aggiunto `GET /circles` con flag `isAlreadyMember` per-user — prerequisito funzionale minimo senza espandere scope.

---

## Soluzione tecnica

**Approccio:** Tre nuovi endpoint in `CircleEndpoints.cs`. Zero modifiche al modello dati o migrazioni — struttura già in DB da US-002. Frontend: nuovo componente `BrowseCirclesComponent` + tre metodi su `CircleService`.

**Decisioni chiave:**
- `GET /circles` espone `isAlreadyMember` con LEFT JOIN su `CircleMemberships` per l'utente corrente — semplifica UX senza endpoint separato. Volume MVP irrilevante.
- `GET /circles/{id}/members` accessibile da qualsiasi utente autenticato — zero logica di ruolo per MVP.
- Join su `IsPrivate=true` ritorna 403 — non rompe infrastruttura futura.

**Componenti toccati:**

| Componente | Modifica |
|---|---|
| `src/Golp.Api/Endpoints/CircleEndpoints.cs` | +3 endpoint, +2 record DTO |
| `src/Golp.Tests/Integration/CircleIntegrationTests.cs` | +test join e members (nuova classe o extend) |
| `frontend/golp-app/src/app/circles/circle.service.ts` | +3 metodi, +2 interface |
| `frontend/golp-app/src/app/circles/my-circles/my-circles.component.html` | link "Scopri circoli" |
| `frontend/golp-app/src/app/circles/browse-circles/` | nuovo componente (lista + join) |
| `frontend/golp-app/src/app/app.routes.ts` | +route `/browse-circles` |

**Contratti API:**

```
GET /circles
Authorization: Bearer
→ 200: [{id, name, sport, memberCount, isAlreadyMember}]

POST /circles/{id}/join
Authorization: Bearer
→ 200: {circleId, myRating: 1000}
→ 403: {error: "Il circolo è privato"}  // IsPrivate=true
→ 404: {error: "Circolo non trovato"}
→ 409: {error: "Sei già membro di questo circolo"}

GET /circles/{id}/members
Authorization: Bearer
→ 200: [{userId, name, rating, rank}]
→ 404: {error: "Circolo non trovato"}
```
