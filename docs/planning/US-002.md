## Analisi requisiti

**Flusso principale:** utente autenticato crea un circolo scegliendo nome + sport. Il backend assegna automaticamente la config sport (`point_unit`, `sets`, `team_size=2`) dalla lista hardcoded. Il creatore diventa automaticamente membro con rating iniziale 1000.

**Edge case critici:** (1) unicità nome circolo è per-creatore, non globale — due utenti diversi possono avere un circolo "Padel Roma"; (2) `sport` non in lista → 400 esplicito; (3) il rating 1000 vive su `CircleMembership`, non su `User` — è per-circolo by design; (4) AC5 "isolamento" non è validazione del singolo endpoint ma invariante di tutte le query future — garantito filtrando sempre per `circle_id`.

**Open question PRD risolta:** MVP creazione libera (open). Modello dati include subito `IsPrivate` (default false) + `JoinCode` (nullable) per abilitare la modalità privata in future senza migrazioni. Tutti i circoli nel MVP sono `IsPrivate=false`.

**Requisiti impliciti:** JWT richiesto su POST /circles e GET /circles/me (userId estratto dal claim `sub`). `GET /sports` è pubblico (no auth). `team_size` fisso a 2 in tutti i sport del MVP.

---

## Soluzione tecnica

**Approccio:** due nuove tabelle (`Circles`, `CircleMemberships`) + tre endpoint (`POST /circles`, `GET /circles/me`, `GET /sports`). Sport config hardcoded in classe statica `SportsConfig` — unica fonte di verità, evita desync tra DB e codice. `CircleMembership` è la tabella pivot User↔Circle che conterrà anche il rating (usato nelle storie successive).

**Componenti toccati:**

| Componente | Tipo | Descrizione |
|---|---|---|
| `Circle` | Entità | Id, OwnerId FK→Users, Name, Sport, PointUnit, Sets, TeamSize, IsPrivate, JoinCode?, CreatedAt, UpdatedAt? |
| `CircleMembership` | Entità | Id, CircleId FK→Circles, UserId FK→Users, Rating (default 1000), JoinedAt |
| `AppDbContext` | EF Core | +DbSet<Circle>, +DbSet<CircleMembership>, +configurazione unique indexes |
| `SportsConfig` | Classe statica | Lista hardcoded 4 sport config — fonte di verità |
| `CircleEndpoints` | Minimal API | POST /circles, GET /circles/me, GET /sports |
| `CircleService` | Angular service | HTTP calls: createCircle(), getMyCircles(), getSports() |
| `CreateCircleComponent` | Angular | Form nome + select sport popolato da GET /sports |
| `MyCirclesComponent` | Angular | Lista circoli con nome, sport, rating, memberCount |

**Modello dati:**
```sql
CREATE TABLE Circles (
  Id          UNIQUEIDENTIFIER PRIMARY KEY DEFAULT NEWID(),
  OwnerId     UNIQUEIDENTIFIER NOT NULL REFERENCES Users(Id),
  Name        NVARCHAR(100) NOT NULL,
  Sport       NVARCHAR(50) NOT NULL,
  PointUnit   NVARCHAR(20) NOT NULL,
  Sets        BIT NOT NULL DEFAULT 0,
  TeamSize    INT NOT NULL DEFAULT 2,
  IsPrivate   BIT NOT NULL DEFAULT 0,
  JoinCode    NVARCHAR(20) NULL,
  CreatedAt   DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
  UpdatedAt   DATETIME2 NULL
);
-- UNIQUE INDEX su (OwnerId, Name)

CREATE TABLE CircleMemberships (
  Id        UNIQUEIDENTIFIER PRIMARY KEY DEFAULT NEWID(),
  CircleId  UNIQUEIDENTIFIER NOT NULL REFERENCES Circles(Id) ON DELETE CASCADE,
  UserId    UNIQUEIDENTIFIER NOT NULL REFERENCES Users(Id),
  Rating    INT NOT NULL DEFAULT 1000,
  JoinedAt  DATETIME2 NOT NULL DEFAULT GETUTCDATE()
);
-- UNIQUE INDEX su (CircleId, UserId)
```

**API contracts:**
```
GET /sports
  200: [{ sport, pointUnit, sets, teamSize }]

POST /circles          [Auth: Bearer JWT]
  Body: { name, sport }
  200: { id, name, sport, pointUnit, sets, teamSize, joinCode?, memberCount: 1 }
  400: { error }   // nome vuoto/troppo lungo, sport non valido
  409: { error }   // nome duplicato per stesso creatore

GET /circles/me        [Auth: Bearer JWT]
  200: [{ id, name, sport, memberCount, myRating, myRank }]
  // myRank: posizione 1-based dell'utente nella classifica del circolo (rank per rating DESC)
```

**Sport config hardcoded:**
```json
[
  { "sport": "padel",       "pointUnit": "games",  "sets": true,  "teamSize": 2 },
  { "sport": "beachtennis", "pointUnit": "games",  "sets": true,  "teamSize": 2 },
  { "sport": "basket2v2",   "pointUnit": "points", "sets": false, "teamSize": 2 },
  { "sport": "burraco",     "pointUnit": "score",  "sets": false, "teamSize": 2 }
]
```

**Rischi:** nessun rischio bloccante. Pattern JWT claim extraction (`ClaimTypes.NameIdentifier`) già validato in US-001.
