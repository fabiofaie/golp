## Analisi requisiti

La `.brand` class in `src/styles.scss` usa `var(--color-accent)` per il colore del testo "GOLP". `--color-accent` è `#FF5500` ed è condivisa con bottoni, focus ring e altri elementi UI — NON va cambiata globalmente. Serve una variabile CSS dedicata `--color-brand` applicata solo al logo.

Il cambio colore deve essere compile-time (guidato da `environment.ts`), non runtime. La tecnica corretta in Angular è: valore costante in `environment.ts` → `AppComponent.ngOnInit()` lo legge e lo imposta come CSS custom property su `:root` via `document.documentElement.style.setProperty`. Il bundle include già il valore corretto per ogni build — nessun check runtime su hostname.

Per il manifest: `public/manifest.webmanifest` è un asset statico copiato com'è. Non è possibile templarlo con Angular. Soluzione: tre manifest separati (`manifest.webmanifest`, `manifest.testing.webmanifest`, `manifest.development.webmanifest`) con path icona distinti; tre `src/index.ENV.html` che linkano il manifest giusto; `angular.json` usa l'opzione `"index"` per-configurazione.

Nota: `environment.prod.ts` è usato dalla configurazione `testing` (API test, non prod) — aggiungere `brandColor` giallo a quel file è sufficiente, non serve un quarto environment file.

## Soluzione tecnica

**Logo color:** Aggiungere `brandColor: string` ai tre file environment. `AppComponent.ngOnInit()` inietta `DOCUMENT` e chiama `document.documentElement.style.setProperty('--color-brand', environment.brandColor)`. `styles.scss` aggiunge `--color-brand: #FF5500` come fallback in `:root` e cambia `.brand { color: var(--color-brand); }`.

**Manifest per-env:** Tre manifest, tre index.html, due `"index"` override in `angular.json`. Il glob `**/*` in `public` copia tutte le cartelle icona (`icons/`, `icons-test/`, `icons-dev/`) in ogni build — nessun cambio agli `assets` per-configurazione.
