## Analisi requisiti

**AC e comportamenti concreti:**
1. **Classifica ordinata per rating decrescente** — nuovo endpoint `GET /circles/{id}/leaderboard` restituisce due gruppi: `classified` (≥1 partita confirmed nel circolo) ordinata per `rating DESC, confirmedMatches DESC`; `unclassified` (0 partite confirmed), lista senza rank.
2. **Aggiornamento subito dopo conferma** — "subito" = al prossimo caricamento della schermata (refresh su navigazione). Nessuna infra realtime richiesta.
3. **Posizione current user evidenziata** — il frontend confronta ogni `userId` della lista con `AuthService.currentUserId`; applica CSS `leaderboard-row--current`. Nessuna logica aggiuntiva backend.
4. **Ordinamento secondario deterministico** — a parità di rating: `confirmedMatches DESC` (chi ha giocato di più è avvantaggiato).
5. **Giocatori 0 partite → sezione "Non classificati"** — non appaiono nella classifica numerata; sezione separata sotto con solo nome.
6. **Cambio circolo → classifica del circolo selezionato** — il componente legge `:circleId` da route param e chiama l'endpoint con quel valore.

**Edge case:**
- Circolo appena creato: tutti a 0 confirmed → `classified` vuota, `unclassified` con tutti i membri.
- Current user in `unclassified` → deve comunque essere evidenziato nella sezione Non classificati.
- Endpoint esistente `GET /circles/{id}/members` (US-003) rimane invariato.

**Requisiti impliciti:**
- Nuovo endpoint `GET /circles/{id}/leaderboard` distinto dal members.
- La query confirmed-matches-per-player richiede join su `Matches` (4 player column) con `Status = "confirmed"`.
- Route Angular nuova: `/circles/:circleId/leaderboard`; link "Classifica" da ogni card in MyCircles.

## Soluzione tecnica

**Approccio:** Nuovo handler `GetCircleLeaderboardAsync` aggiunto a `CircleEndpoints`. Query EF Core con `SelectMany` sulle 4 colonne player di `Matches` per contare le partite confirmed per-player. Frontend: `CircleLeaderboardComponent` standalone alla route `/circles/:circleId/leaderboard`, dato servito da `CircleService.getLeaderboard()`.

**Componenti toccati:**

| Layer | Componente | Modifica |
|-------|-----------|----------|
| Backend | `CircleEndpoints.cs` | aggiunge `GetCircleLeaderboardAsync` + MapGet `/circles/{id}/leaderboard` |
| Frontend | `circle.service.ts` | aggiunge `LeaderboardEntry`, `LeaderboardUnclassified`, `LeaderboardResponse`, `getLeaderboard()` |
| Frontend | `circle-leaderboard/` | **new component** `CircleLeaderboardComponent` (ts + html + scss) |
| Frontend | `app.routes.ts` | aggiunge route `/circles/:circleId/leaderboard` |
| Frontend | `my-circles.component.html` | aggiunge link "Classifica" per ogni card |

**Contratto API:**

```
GET /circles/{id}/leaderboard
Authorization: Bearer <jwt>
→ 200 {
    "classified": [
      { "userId": "...", "name": "...", "rating": 1050, "rank": 1, "confirmedMatches": 5 }
    ],
    "unclassified": [
      { "userId": "...", "name": "..." }
    ]
  }
→ 404  circolo non trovato
→ 401  non autenticato
```

**Query backend (schema LINQ):**
```csharp
var confirmedCounts = await db.Matches
    .Where(m => m.CircleId == id && m.Status == "confirmed")
    .SelectMany(m => new[] { m.Team1Player1Id, m.Team1Player2Id, m.Team2Player1Id, m.Team2Player2Id })
    .GroupBy(uid => uid)
    .Select(g => new { UserId = g.Key, Count = g.Count() })
    .ToDictionaryAsync(x => x.UserId, x => x.Count);

var members = await db.CircleMemberships
    .Where(m => m.CircleId == id)
    .Select(m => new { m.UserId, Name = m.User.Name, m.Rating })
    .ToListAsync();

var classified = members
    .Where(m => confirmedCounts.ContainsKey(m.UserId))
    .OrderByDescending(m => m.Rating)
    .ThenByDescending(m => confirmedCounts[m.UserId])
    .Select((m, i) => new { userId = m.UserId, name = m.Name, rating = m.Rating, rank = i + 1, confirmedMatches = confirmedCounts[m.UserId] })
    .ToList();

var unclassified = members
    .Where(m => !confirmedCounts.ContainsKey(m.UserId))
    .Select(m => new { userId = m.UserId, name = m.Name })
    .ToList();
```

**Decisioni chiave:**
- **Endpoint separato da GET /circles/{id}/members** — semantica diversa; members serve il browse, leaderboard serve la classifica con confirmed split.
- **isCurrentUser in frontend** — evita di portare il userId nel response (già disponibile in AuthService).
- **confirmedMatches calcolato in DB** — non in memoria, query su Matches filtrata per circolo e status.

**Rischi:**
- US-007 non DONE → tutti a rating 1000, classifica poco significativa (ma funzionale).
- La query `SelectMany` su 4 colonne è un pattern insolito in EF — testare performance con dataset medio.

## Rischi aperti

- **Query SelectMany su 4 colonne**: pattern non standard in EF Core — verificare che produca SQL efficiente, potrebbe richiedere un indice su Matches(CircleId, Status).
